01 — The Problem

You build something great in Claude. Then what?

  • →Copy it into a Google Doc, format it again, share a link with edit access you didn't mean to give
  • →Export as PDF, attach to an email, client asks for one change, do it all over
  • →Screenshot it, paste it in Canva, spend 20 minutes making it look like what Claude already made
  • →Leave Claude, open three other tools, lose the thread entirely
02 — What This Is

A publishing layer on top of Claude.

Anything you build with HTML/CSS goes live as a real hosted link instantly. You never leave the interface.

Proposals Presentations Mini Sites Reference Docs One-Pagers Client Deliverables Landing Pages Anything HTML/CSS
03 — How It Works

The whole loop, never leaving Claude.

1

Describe what you want

Tell Claude what to build. Proposal, mini site, reference doc, presentation. It designs it in the conversation.

2

Review and approve

Claude shows you the design as an artifact. You review it, request changes, iterate until it's right.

3

Say "ship it"

Claude pushes the file to GitHub. Railway deploys it automatically in about 30 seconds.

4

There's a URL

Claude returns a live link. You copy it and send it. Done. No other tools touched.

04 — Under the Hood

Three pieces. All connected.

Where you work

Claude + Skill Doc

A skill document tells Claude exactly how to publish: slug format, repo name, your Railway URL, your design system. Claude handles the rest.

Where files live

GitHub Repo

Every published file lives here as a standalone HTML file. Each gets a unique slug. All permanent. All version controlled.

Where they're served

Railway

Auto-deploys on every GitHub push. Your files are live at a real URL within 30 seconds of saying ship it. No manual deploys ever.

Step 01 — GitHub Repo

Create your GitHub repo.

1
Go to github.com and sign in. If you don't have an account, create one now.
2
Click the + icon in the top right corner, then click New repository.
3
Name it publishing-server. Lowercase, no spaces.
4
Set visibility to Private.
5
Check Add a README file. This is required — Claude cannot push to an empty repo.
6
Click Create repository. Note your username and repo name — you'll give these to Claude next.

Step 03 — Railway

Deploy your server to Railway.

1
Go to railway.app and sign up with your GitHub account. This links the two automatically.
2
Click New Project, then Deploy from GitHub repo. Select publishing-server.
3
Railway detects Node automatically and starts the build. Go to the Deployments tab and watch the logs. You're looking for: Proposals server running on port 8080
4
Go to Settings → Networking and click Generate Domain. Copy the URL — this is your base URL.
5
Go to Settings → Source. If you see a Disable button next to auto-deploy, it's already on. You're done with Railway forever.

Step 04 — Skill Document

Customize and install your skill doc.

1
Copy the customization prompt below and paste it into a new Claude conversation.
2
Claude will ask for your GitHub username, Railway URL, and optionally your brand colors and fonts. Answer each question.
3
Claude outputs your fully customized skill doc. Copy the entire thing.
4
In Claude, go to Settings → Custom Instructions. Paste the skill doc in and save.
5
Start a new conversation. Ask Claude to build something. Say ship it. Your live URL comes back in chat.
