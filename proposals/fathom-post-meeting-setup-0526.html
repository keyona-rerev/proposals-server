Key difference from the Google Meet build

The Google Meet build (T018) watches a Drive folder for new docs. This build polls the Fathom API directly — no Drive folder required. Fathom provides richer data (speaker-matched transcripts, AI summaries, recording URLs), which makes for better follow-ups. The tradeoff: you need a Fathom account and API key.

Before we start

What You Need

Checklist
  • A Fathom account with at least one recorded meeting — free tier works. Download the Fathom desktop app and record a call to test with.
  • A Fathom API key — get one at app.fathom.video → Settings → Integrations → API. This is different from your login credentials.
  • An Anthropic API key — get one at console.anthropic.com. Free to sign up, pay-as-you-go.
  • A Google account with Gmail access — where drafts and digest emails will land.
  • A browser tab open to script.google.com — nothing to install.
🖥️ During this session

Step 1

1Get Your Fathom API Key

  • Log into app.fathom.video

  • Click your profile icon → Settings

  • Find the Integrations or API section

  • Click Create API Key (or copy an existing one)

  • Save it somewhere safe — you'll paste it into Script Properties in Step 3

Keep this key private

Your Fathom API key gives access to all your meeting recordings. Never paste it directly into script code — always store it in Script Properties.

Step 2

2Create the Apps Script Project

  • In a new tab, go to script.google.com

  • Click New Project — name it Fathom Post-Meeting Engine

  • You'll see a default Code.gs file — you'll be replacing it with the 9 files below

File naming matters

Click the + icon next to "Files" to add each new file. Select Script for .gs files, HTML for Dashboard. Names are case-sensitive.

About STAGING_EMAIL

When STAGING_EMAIL is set, all drafts and digest emails go to that address instead of the real recipients. Use it during setup to verify output before going live. Clear it when you're ready to send for real.

Step 5

5Authorize the Script

About the "unsafe" warning

Standard for personal scripts. You're authorizing access to your own Gmail and Google Calendar — nothing external except Fathom and the Anthropic API.

MeetingProcessor.gs Core engine — orchestrates Fathom fetch, Claude call, output dispatch
✏ Customize prompt
▼
What to change — 1 spot

The only section you edit is the Claude system prompt inside _callClaude() — specifically the systemPrompt variable that describes who you are, your company, and your email style.

Do not touch: run(), backfill(), reprocessById(), _processMeeting(), _callClaude() logic outside the system prompt. These handle meeting routing and output dispatch.

Use the Claude Customization Prompt in the next section to generate your version of the system prompt.

const MeetingProcessor = {

  run() {
    if (!CONFIG.FATHOM_API_KEY) return 'FATHOM_API_KEY not set in Script Properties.';
    if (!CONFIG.ANTHROPIC_API_KEY) return 'ANTHROPIC_API_KEY not set in Script Properties.';
    const meetings = FathomClient.getNewMeetings();
    if (!meetings.length) { Logger.log('No new meetings.'); return 'No new meetings to process.'; }
    let processed = 0;
    const now = new Date().toISOString();
    meetings.forEach(meeting => {
      try { Logger.log('Processing: ' + meeting.title); this._processMeeting(meeting); processed++; }
      catch(e) { Logger.log('Error: ' + meeting.title + ': ' + e.message); ProcessingLog.markSkipped(meeting.id, meeting.title, e.message); }
    });
    CONFIG.LAST_RUN_TIMESTAMP = now;
    return 'Processed ' + processed + ' of ' + meetings.length + ' meeting(s).';
  },

  backfill(limit) {
    limit = limit || 100;
    if (!CONFIG.FATHOM_API_KEY) return 'FATHOM_API_KEY not set.';
    const meetings = FathomClient.getAllMeetings(limit);
    if (!meetings.length) return 'No meetings found in Fathom.';
    const existing = ProcessingLog.getProcessedIds();
    let added = 0;
    meetings.forEach(meeting => {
      if (existing.has(meeting.id)) return;
      const attendeeStr = meeting.attendees.map(a => a.name || a.email).filter(Boolean).join(', ') || 'Unknown';
      ProcessingLog.markBackfilled(meeting.id, meeting.title, attendeeStr, meeting.fathomUrl);
      added++;
    });
    return 'Loaded ' + added + ' meetings (' + (meetings.length - added) + ' already existed). Select any to reprocess.';
  },

  reprocessById(recordingId) {
    const meeting = FathomClient.getMeetingById(recordingId);
    if (!meeting) throw new Error('Could not fetch meeting ' + recordingId + ' from Fathom.');
    this._processMeeting(meeting);
    return 'Reprocessed: ' + meeting.title;
  },

  _processMeeting(meeting) {
    if (!meeting.hasContent && !meeting.hasTranscript && !meeting.summary) {
      ProcessingLog.markSkipped(meeting.id, meeting.title, 'No transcript or summary available');
      return;
    }
    const { result, rawResponse } = this._callClaude(meeting);
    if (!result) {
      ProcessingLog.markSkipped(meeting.id, meeting.title, 'Claude parse failed: ' + (rawResponse||'').substring(0,200));
      return;
    }
    const { follow_up_draft } = result;
    if (follow_up_draft && meeting.attendees.length) {
      const allEmails = meeting.attendees.map(a => a.email).filter(Boolean);
      if (allEmails.length && !follow_up_draft.to) follow_up_draft.to = allEmails.join(', ');
    }
    const draftId = OutputDispatcher.dispatchDraft(follow_up_draft, meeting.attendees, meeting.title);
    OutputDispatcher.sendDigest({ meetingName: meeting.title, fathomUrl: meeting.fathomUrl, attendees: meeting.attendees, draftId });
    ProcessingLog.markProcessed(meeting.id, meeting.title, meeting.attendees.map(a => a.name||a.email).join(', '), draftId, meeting.fathomUrl);
  },

  _callClaude(meeting) {
    const attendeeList = meeting.attendees.length
      ? meeting.attendees.map(a => a.name+(a.email?' ('+a.email+')':'')).join(', ')
      : 'Unknown attendees';
    const meetingDate = new Date(meeting.createdAt);
    const meetingDateStr = meetingDate.toLocaleDateString('en-US',{weekday:'long',year:'numeric',month:'long',day:'numeric'});
    const now = new Date();
    const sendDateStr = now.toLocaleDateString('en-US',{weekday:'long',year:'numeric',month:'long',day:'numeric'});
    const sameDay = meetingDate.toDateString() === now.toDateString();
    const daysDiff = Math.floor((now-meetingDate)/(1000*60*60*24));
    const isGroup = meeting.attendees.filter(a => a.email).length > 1;
    const timeContext = 'TIME CONTEXT:\n- Meeting date: '+meetingDateStr+'\n- Email send date: '+sendDateStr+'\n- Time since meeting: '+(sameDay?'same day':daysDiff+' day(s) ago')+'\n\nTIME RULES:\n1. NEVER use relative time words\n2. Use the actual date or a neutral opener\n3. Specific future deadlines are fine\n4. When in doubt, omit the time reference';
    const fathomRef = meeting.fathomUrl ? '\nFathom recording: '+meeting.fathomUrl : '';
    const emailStyle = isGroup
      ? 'EMAIL STYLE: group reply-all. Address everyone by first name. Warm, direct, specific. No generic openers. No emojis. Plain text only. Close with "Warmly," then "YOUR FIRST NAME" on the next line.'+fathomRef
      : 'EMAIL STYLE: warm, direct, specific. No generic openers. No emojis. Plain text only. Lead with something concrete. Close with "Warmly," then "YOUR FIRST NAME" on the next line.'+fathomRef;

    // ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
    // CUSTOMIZE THIS BLOCK — use the Claude Prompt below to generate
    // ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
    const systemPrompt =
      'You are a post-meeting intelligence assistant for [YOUR NAME].\n\n'+
      'ABOUT [YOUR NAME] AND [YOUR COMPANY]:\n'+
      '- [Company name]: [one paragraph description of what you do]\n'+
      '- [Your role]: [what you lead and who you serve]\n'+
      '- Your network spans: [key stakeholder types]\n\n'+
      'CONTACT INTELLIGENCE: For each attendee identify likely role: [list your role types]\n\n'
      + emailStyle + '\n\n' + timeContext
      + '\n\nReturn ONLY raw JSON. No markdown. No preamble.\nSTRUCTURE: {"follow_up_draft":{"to":"string","subject":"string","body":"string"}}';
    // ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

    const content = meeting.summary
      ? 'Meeting: '+meeting.title+'\nAttendees: '+attendeeList+'\n\nFATHOM SUMMARY:\n'+meeting.summary+(meeting.transcriptText?'\n\nTRANSCRIPT EXCERPT:\n'+meeting.transcriptText.substring(0,8000):'')
      : 'Meeting: '+meeting.title+'\nAttendees: '+attendeeList+'\n\nTRANSCRIPT:\n'+meeting.transcriptText;
    const payload = { model: 'claude-sonnet-4-20250514', max_tokens: 1500, system: systemPrompt, messages: [{ role: 'user', content }] };
    try {
      const response = UrlFetchApp.fetch('https://api.anthropic.com/v1/messages', { method: 'post', contentType: 'application/json', headers: { 'x-api-key': CONFIG.ANTHROPIC_API_KEY, 'anthropic-version': '2023-06-01' }, payload: JSON.stringify(payload), muteHttpExceptions: true });
      const data = JSON.parse(response.getContentText());
      if (!data.content || !data.content[0]) return { result: null, rawResponse: response.getContentText().substring(0,200) };
      const raw = data.content[0].text.trim();
      try { return { result: JSON.parse(raw), rawResponse: raw }; }
      catch(e) { try { return { result: JSON.parse(raw.replace(/^```json\n?/,'').replace(/```$/,'').trim()), rawResponse: raw }; } catch(e2) { return { result: null, rawResponse: raw }; } }
    } catch(e) { return { result: null, rawResponse: e.message }; }
  }
};

If something goes wrong

Troubleshooting

"No new meetings" every time the engine runs
The script only fetches meetings created after LAST_RUN_TIMESTAMP. Use the dashboard → "Load Past Meetings" (serverBackfill) to pull existing Fathom meetings into the log, then select any and click Reprocess.
Fathom API returns 401 or 403
Your FATHOM_API_KEY in Script Properties is incorrect or expired. Go to app.fathom.video → Settings → API and regenerate. Update Script Properties and re-run pingFathom().
Draft created but attendee email is missing or wrong
Fathom tries to match speakers to calendar invitees. If it can't match, the script falls back to Google Calendar. Check the Execution Log for "Attendees from transcript: 0" — this means matching failed. Ensure your Google Calendar event was invite-based.
Claude parse failed
Check the Execution Log for the raw Claude response. Usually a malformed systemPrompt. Run pingClaude() to confirm the API key works, then check for unescaped quotes in MeetingProcessor.gs.
I want to test without sending to real attendees
Set STAGING_EMAIL in Script Properties to your own address. All drafts and digests will redirect there until you clear it.
The follow-up is too generic / doesn't reference the meeting
Open the dashboard → Run Diagnostic to see what data Fathom returned. If the summary is thin, check that Fathom's AI summary feature is enabled in your account settings.
